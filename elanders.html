Customization

Batch Processing

In Elanders solution, we have modified the batch process to run directly out of Veracore, using the "Wave PPU" as the batch key.Because of this, the following steps must be taken to diagnose/test the process. You must be signed in as one of their users, and be linked to one of the stations from their list.

The standard batch process runs similar to the normal process, in which they select a batch key, enter some default parameters, and press Process. Once the process starts, the system will pull in a list of package IDs (order numbers) from Veracore based on the selected wave PPU, and process them through one by one.

Depending on if you had the Batch PDF option selected on the process screen, the system will either print each label as it is processed (not checked), or will consolidate the labels into 1 PDF file for the user to print (if checked).

Once the shipping side of things is complete, the system will then send a set of update/insert commands to Veracore to close the packages. This is done in groups of 50 as a batch insert for speed. The Update Veracore by Wave feature performs this same process by itself. The Reprint batch feature will either open the existing PDF (if on the server), or will recreate the PDF, and send any necessary labels to the printer. Batches are typically pre-printed before packing starts.

Rateshops

In Elanders solution, most/all rateshops do not actually shop for rates. Instead, they are essentially business rules that vary from client to client (use OMS ID/udf1 as identifier). They refer to this as their "Ground Best Way logic". If Elanders ever requests to update this logic follow the steps below.

  1. Save the updated files they send you to our ARE-Clients folder @ \Elanders Americas\GRDBW. The files need to match the formats of the other Excel files found in this folder.
  2. Once saved, run the Elanders.RateFileBuilder.exe file found at E:\Projects\SHIPSTORE\v3\ShipStore Enterprise\Elanders\Extra Utilities\RateAndZoneFileBuilder\bin\Debug. (If you don't see the file, you'll need to build the solution.)
  3. Once the app is open, click the "GRDBW Routing File" tab.
  4. In the top box, Select one of the files saved in step 1.
  5. In the bottom box, select the matching .JS file from the same folder in the above step (i.e. Ounce tables = LW, Pound tables = GRDBW)
  6. Click Generate once the 2 file paths are entered.
  7. Repeat steps 4-6 for each files (should be 2 max)
  8. Once the files are generated, open the Elanders ShipStore Website solution found at E:\Projects\SHIPSTORE\v4\ShipStore Enterprise\Elanders\ShipStore Website
  9. Copy/Paste the updated .JS files generated above to the Resources folder of the website project
  10. Rebuild, then publish to production servers
